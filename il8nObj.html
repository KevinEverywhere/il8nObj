{{'locale_example' | il8nObj:'client0':'ClientProducts':'ProductName'}}

{{'locale_example' | il8nObj:'client1':'Client'}}

{{'locale_example' | il8nObj:'client1':'ClientProducts':'ProductName'}}

The text above comes from the file il8nObj.js. The structure for drilling down into the Object is: 

     {{'filePrefix' | il8nObj:'keyValue':'property':'subproperty':'subsubproperty'}}

So, the top example above is: 

     {{'locale_example' | il8nObj:'intro'}}

This refers to "Introduction to Internationalization" from returning the value from the clients key in the file at locale_example_en.json. The fifth example drills down into the client0 value to its ClientProducts.ProductName property.: 

     {{'locale_example' | il8nObj:'client0':'ClientProducts':'ProductName'}}

I built this filter, because I wanted to map a simple hierarchy to a number of languages. There was no exisitng filter that I could find to achieve a structured mapping, so this is what I came up with. I hope this can help someone out of a similar situation.
